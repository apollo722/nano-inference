# Nano-Inference: Collaborative Engineering & Learning Mandate

This project follows a "Learning-First" implementation model. All agents must adhere to these rules:

1. **No Code Dumps:** Never implement or provide large blocks of code without first explaining the underlying design rationale and the "Why" behind specific choices.
2. **Research & Design First:** Before any implementation, provide a detailed design rationale and an implementation plan. Wait for user review and confirmation.
3. **Collaborative Implementation:** The user may choose to implement the code themselves after the design phase. The agent's role is to review, provide feedback, and ensure alignment with the project's architecture.
4. **Teaching & Pair Programming:** Treat every task as a pair-programming session. Focus on knowledge transfer and ensuring the user understands every architectural decision.
5. **Contextual Integrity:** Always align changes with the `PLAN.MD` and the existing architectural patterns (ABC-first, Factory patterns, CPU/GPU separation).
